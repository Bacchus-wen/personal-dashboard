# 我的作品管理功能设计

## 目标

为 Theodore 个人作品集增加一套真实可维护的“我的作品”功能。管理员可以在受保护的后台创建、编辑、预览、发布、排序和删除作品；访客可以通过 `/works` 浏览公开作品，并进入独立详情页了解项目和访问项目网站。

本流程只负责作品管理、公开展示和详情页。优秀开源项目、首页精选作品展示、真实图片上传和 Supabase Storage 管理分别留到后续流程。

## 范围

### 本流程包含

- 新增公开作品列表 `/works` 和作品详情 `/works/[slug]`。
- 将导航中的“我的项目”改为“我的作品”，并指向 `/works`。
- 保留 `/projects` 路由，供后续“优秀项目”流程使用。
- 新增后台作品列表、新建、编辑、草稿预览和回收站。
- 支持封面、多张详情截图、Markdown 详细介绍、技术标签、项目状态、日期、外部网站和 GitHub 链接。
- 支持公开列表按技术标签与项目状态筛选。
- 支持管理员定义作品顺序和精选标记。
- 使用现有管理员登录、服务端数据访问、设计系统和 Markdown 渲染能力。

### 本流程不包含

- 不迁移当前 `/projects` 的四条演示数据。
- 不在首页展示精选作品；本流程只保存精选标记。
- 不建设优秀 GitHub 项目页面。
- 不提供真实文件上传、裁剪或 Supabase Storage 管理。
- 不自动检查项目网站是否可访问。
- 不提供作品分类、批量操作、版本历史、协作或评论。
- 不安装新依赖，不使用 Docker。

## 数据与安全架构

采用两个职责独立的业务表：

### `works`

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| `id` | UUID | 主键 |
| `name` | Text | 必填，去除首尾空白后不可为空 |
| `slug` | Text / Null | 草稿、私密和归档状态可为空；公开时必填且唯一 |
| `summary` | Text / Null | 简短纯文本摘要；公开时必填 |
| `description` | Text / Null | Markdown 详细介绍；公开时必填 |
| `cover_path` | Text / Null | 项目本地路径或 HTTPS URL |
| `tech_stack` | Text Array | 去重后的技术标签列表 |
| `status` | Enum | `developing`、`maintained`、`completed`、`stopped` |
| `visibility` | Enum | `draft`、`private`、`public`、`archived` |
| `started_on` | Date / Null | 开始日期 |
| `completed_on` | Date / Null | 完成日期，不得早于开始日期 |
| `website_url` | Text / Null | 公开时必填，仅允许 HTTPS |
| `github_url` | Text / Null | 可选，仅允许 HTTPS |
| `website_available` | Boolean | 管理员手动标记项目网站是否可访问 |
| `featured` | Boolean | 为后续首页精选展示保留 |
| `sort_order` | Integer | 非负整数，数值越小越靠前 |
| `seo_title` | Text / Null | 可选详情页 SEO 标题 |
| `seo_description` | Text / Null | 可选详情页 SEO 描述 |
| `seo_image_path` | Text / Null | 可选项目本地路径或 HTTPS URL |
| `deleted_at` | Timestamptz / Null | 有值时位于回收站 |
| `created_at` | Timestamptz | 创建时间 |
| `updated_at` | Timestamptz | 最后修改时间 |

新作品默认值：

- `visibility = draft`
- `status = developing`
- `website_available = true`
- `featured = false`
- `sort_order = 0`

草稿只要求填写名称。草稿、私密和归档作品允许暂时没有 slug；转为公开时，必须补充有效且唯一的 slug、摘要、详细介绍和 HTTPS 项目网站。

slug 仅允许小写英文字母、数字和连字符，不能以连字符开头或结尾。公开详情地址使用 `/works/[slug]`。没有 slug 的草稿通过受保护的 ID 路由 `/admin/works/[id]/preview` 预览，不生成或暴露公共详情地址。

### `work_screenshots`

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| `id` | UUID | 主键 |
| `work_id` | UUID | 关联作品；永久删除作品时级联删除 |
| `image_path` | Text | 必须为项目本地路径或 HTTPS URL |
| `caption` | Text / Null | 可选图片说明 |
| `sort_order` | Integer | 非负整数，同一作品内不可重复 |
| `created_at` | Timestamptz | 创建时间 |
| `updated_at` | Timestamptz | 最后修改时间 |

### 安全边界

- `works` 和 `work_screenshots` 启用 RLS。
- 撤销 `anon` 与普通 `authenticated` 对业务表的直接权限。
- 仅向 `service_role` 授予服务端需要的权限。
- 公开页面由 Next.js 服务端读取，并强制应用 `visibility = public`、`deleted_at is null` 和有效 slug 条件。
- 所有后台写入操作在服务端重新验证当前登录用户是唯一管理员。
- 浏览器不能直接读取或修改作品业务表。
- Markdown 禁用内嵌 HTML，并复用现有安全渲染组件。
- 所有链接和图片路径在服务端验证；错误响应不得泄露密钥、数据库连接信息或内部堆栈。

## 公开页面

### 作品列表 `/works`

- 页面标题为“我的作品”。
- 桌面端使用响应式双列布局，移动端使用单列布局，并支持 320px 宽度。
- 卡片采用已确认的 A+B 融合视觉方向：
  - 大幅封面或浏览器窗口式网站预览作为主要视觉。
  - 信息区使用分层排版，展示名称、摘要、状态、日期、精选标记和技术标签。
  - 保留现有柔和玻璃、圆角、阴影和悬浮抬升效果。
  - 整张卡片是指向 `/works/[slug]` 的语义链接。
- 支持技术标签与项目状态组合筛选，筛选条件记录在 URL 查询参数中。
- 默认先按管理员 `sort_order` 从小到大排列，相同顺序按 `updated_at` 从新到旧排列。
- 没有公开作品时显示明确空状态，不回退到演示数据。
- 数据读取失败时只在作品区域显示可恢复错误状态，不影响网站其他区域。

### 作品详情 `/works/[slug]`

详情页展示：

- 封面、名称、状态、开始与完成日期。
- 简短摘要和技术标签。
- 安全渲染的 Markdown 详细介绍。
- 按 `sort_order` 排列的截图与可选说明。
- 项目网站按钮与可选 GitHub 按钮。
- 独立的 Metadata 标题、描述和分享图片；未单独填写时依次回退到作品名称、摘要和封面。

项目网站和 GitHub 链接使用新标签页打开，并带安全 `rel` 属性。`website_available = false` 时，详情页仍然存在，但项目网站按钮禁用并显示“网站暂不可访问”提示。

不存在、未公开、已归档、已删除或 slug 无效的作品详情统一返回 404，避免泄露后台记录状态。

## 后台页面

### 后台入口

后台首页增加“管理作品”入口，指向 `/admin/works`。所有作品后台路由继续由现有受保护布局统一验证登录状态。

### 作品列表 `/admin/works`

- 支持按名称关键词、项目状态和可见性组合筛选。
- 默认按 `updated_at` 从新到旧展示。
- 每条记录显示名称、状态、可见性、精选标记、顺序和更新时间。
- 提供新建、编辑、预览、移入回收站、进入回收站和调整顺序入口。
- 第一阶段每次只操作一条记录，不提供批量操作。

### 新建与编辑

- `/admin/works/new` 创建作品。
- `/admin/works/[id]/edit` 编辑作品。
- 新建与编辑复用同一作品编辑器组件。
- 编辑器包含所有 `works` 字段和可排序截图列表。
- 技术标签使用自由输入的多标签形式，保存时去除空白、空值和重复项。
- 封面、SEO 图片和截图使用项目本地路径或 HTTPS URL；真实上传延后到流程 5。
- 详细介绍使用现有 Markdown 编辑与预览能力。
- 保存期间禁用重复提交。
- 保存成功后留在当前编辑页并显示短暂成功提示。
- 保存失败时保留输入内容，显示字段级或页面级错误。
- 存在未保存修改时，离开页面前提醒管理员。

### 草稿预览

- `/admin/works/[id]/preview` 使用真实公开详情布局预览任意非回收站作品。
- 预览路由必须验证管理员身份。
- 草稿没有 slug 时仍可正常预览。
- 草稿预览不会改变可见性，也不会向访客公开数据。

### 回收站 `/admin/works/trash`

- 普通删除只写入 `deleted_at`，作品立即停止公开展示。
- 回收站显示已删除作品和删除时间。
- 恢复时清除 `deleted_at`，并强制将可见性改为 `draft`。
- 永久删除前显示“无法恢复”的二次确认；永久删除作品时级联删除截图记录。

## 校验规则

- 名称必填，所有文本字段执行明确长度限制。
- 草稿、私密和归档可暂时没有 slug；公开作品必须拥有有效唯一 slug。
- 公开作品必须拥有名称、摘要、Markdown 详细介绍和 HTTPS 项目网站。
- 技术标签去除首尾空白、空值和重复项，并限制单项长度与总数量。
- 图片字段只允许以 `/` 开头的项目本地路径或 HTTPS URL。
- 项目网站和 GitHub 链接只允许 HTTPS URL。
- 完成日期不得早于开始日期。
- `sort_order` 必须为非负整数。
- 同一作品的截图 `sort_order` 不得重复。
- 精选标记仅保存状态，本流程不影响首页。

## 数据流与缓存

### 公开读取

1. Next.js 服务端接收列表筛选或详情 slug。
2. 服务端仓库应用公开、未删除和有效 slug 条件。
3. 服务端只返回公开页面需要的字段和截图。
4. 列表页应用筛选与排序；详情页按 slug 返回单条作品。
5. 查询失败返回受控错误或 404，不向浏览器发送后台记录。

### 后台写入

1. 管理员提交 Server Action。
2. Server Action 重新验证唯一管理员身份。
3. 服务端规范化并校验输入。
4. 服务端仓库使用 Secret Key 调用数据库事务函数写入作品和截图。
5. 涉及作品与截图的保存必须全部成功或全部失败，避免只保存部分数据。
6. 写入成功后刷新 `/works`、相关详情页和后台作品路由缓存。
7. Server Action 返回成功状态或结构化字段错误。

## 组件边界

- 路由页面保持以服务端组件为主，负责权限检查、查询和组合页面。
- 作品类型、常量、校验、查询规则、仓库接口和服务端仓库分别放在职责清晰的 `src/lib/works/` 文件中。
- 公开作品卡片、筛选器、详情内容和截图画廊放在 `src/components/works/`。
- 后台列表卡片、筛选器、编辑器、截图编辑器和删除操作放在 `src/components/admin/works/`。
- 复用现有 `MarkdownContent`、设计变量、`.glass`、`.card`、`.lift`、`.pill` 和 `.btn`。
- 公开组件不得包含后台操作；后台组件不得绕过 Server Action 直接访问数据库。

## 错误处理

- 数据库尚未执行迁移：公开列表显示空状态或受控错误；后台显示明确迁移提示，不伪装成正常空列表。
- 公开查询失败：只替换作品内容区域，网站导航和其他页面保持可用。
- 后台查询失败：显示错误与重试入口。
- 保存校验失败：保留表单数据并指出具体字段。
- 保存、删除、恢复或永久删除失败：保持原数据状态并显示通用错误。
- 图片加载失败：显示与设计系统一致的占位内容。
- 项目网站被标记不可访问：保留详情页，禁用网站按钮。
- 无权限访问后台或草稿预览：跳转登录页或返回无权限页面。

## 验证策略

### 自动化验证

- 校验测试：
  - 草稿只要求名称。
  - 公开作品拒绝缺少 slug、摘要、详情或 HTTPS 网站。
  - slug 格式、唯一性错误、危险链接和无效图片路径被拒绝。
  - 技术标签去重和限制。
  - 完成日期不得早于开始日期。
- 查询与仓库测试：
  - 公开查询不会返回草稿、私密、归档、已删除或无 slug 数据。
  - 技术标签与状态筛选正确组合。
  - 排序先使用管理员顺序，再使用更新时间。
  - 回收站恢复后强制转为草稿。
  - 作品与截图保存不会产生部分更新。
- 权限测试：
  - 未登录用户和非管理员不能执行写入或草稿预览。
  - 浏览器角色不能直接读取或写入作品业务表。
- UI 行为测试：
  - 技术标签编辑、截图排序、Markdown 预览和未保存离开提醒。
  - 网站不可访问时按钮禁用。
  - 空状态和错误状态正确展示。

### 手动验收

1. 管理员能创建只有名称的草稿，并在后台 ID 预览页查看。
2. 缺少公开必填字段时无法发布，并显示准确错误。
3. 完整作品可公开，并在 `/works` 和 `/works/[slug]` 正确展示。
4. 公共访客无法访问草稿、私密、归档、已删除或不存在的详情。
5. 技术标签和状态筛选正确，URL 可保留筛选状态。
6. 卡片符合已确认的 A+B 融合视觉方向，桌面双列、移动单列，无横向滚动。
7. Markdown、截图顺序、项目网站、GitHub 链接和网站不可访问状态正确。
8. 管理员可以编辑、排序、精选、移入回收站、恢复和永久删除作品。
9. 恢复后的作品自动变为草稿，不会立即重新公开。
10. 没有公开作品时显示空状态，不出现旧演示项目。
11. `npm test`、`npm run lint` 和 `npx tsc --noEmit` 通过。
12. 为减少内存压力，只在最终验收阶段运行一次 `npm run build`。

## 安装与磁盘边界

- 本流程不安装新依赖，不使用 Docker。
- 复用项目现有 Supabase、React Markdown、测试和样式能力。
- 如果实施中发现必须新增依赖，必须先说明用途、预计磁盘占用、默认缓存位置、是否可能写入 C 盘以及无安装替代方案，并等待用户明确批准。
