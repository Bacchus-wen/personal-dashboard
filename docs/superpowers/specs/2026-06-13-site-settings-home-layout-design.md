# 网站设置与首页布局设计

## 目标

为个人博客后台增加可维护的网站设置、社交链接和桌面端首页布局编辑能力。管理员在后台完成修改后，通过一次“保存并发布”统一更新公开首页；未发布的编辑不会影响访客。

本阶段不提供主题颜色编辑、图片上传、布局历史版本或独立预览页面。

## 数据与发布架构

采用三个职责独立的配置表：

- `site_settings`：网站基础信息和首页模块启用状态。
- `social_links`：可排序、可启用的社交链接列表。
- `home_layout`：首页模块的桌面端网格位置和固定尺寸。

后台编辑内容先保存在浏览器页面状态中。管理员点击“保存并发布”后，由受保护的服务端操作统一校验并通过数据库事务写入三个表。发布必须全部成功或全部失败，不允许只更新部分设置。

公开首页仅读取已发布数据。读取失败、数据未初始化或布局无效时，回退到项目内现有内容和默认布局。

## 网站设置

后台 `/admin/settings` 的“网站设置”标签页允许编辑：

- 网站标题
- 用户名
- 状态短语
- 网站描述
- 头像的项目本地路径或 HTTPS URL
- Favicon 的项目本地路径或 HTTPS URL
- 备案号
- 可选备案链接

网站描述同时用作“关于网站”摘要和网页 SEO 描述。备案号为空时，公开页面不显示备案区域。

本阶段不允许修改导航名称、顺序和路由，避免配置错误破坏页面入口。图片真实上传、替换和 Supabase Storage 安全规则延后到流程 5。

## 社交链接

“社交链接”标签页使用灵活列表。每项包含：

- 平台类型
- 预设平台图标
- 显示名称
- 链接
- 排序位置
- 启用状态

管理员可以新增、编辑、排序、启用和删除社交链接。删除前需要确认。链接仅允许 `https://` 和 `mailto:`；未知平台使用通用链接图标。

## 首页布局

“首页布局”标签页提供接近真实首页的桌面端编辑画布：

- 卡片尺寸固定，只允许拖拽位置。
- 位置吸附网格。
- 卡片不能重叠或超出画布。
- 可关闭的模块重新启用后恢复上次保存位置。
- “恢复默认”只更新当前编辑状态，仍需点击“保存并发布”。

核心模块始终启用且不能关闭：

- 侧边导航
- 问候与简历入口
- 社交链接

可选模块：

- 相册
- 时钟
- 日历
- 近日规划
- 随机推荐
- 音乐播放器

移动端不读取桌面坐标，也不提供后台拖拽。移动端按以下固定顺序纵向排列，关闭的可选模块从顺序中移除：

1. 问候与简历入口
2. 社交链接
3. 近日规划
4. 相册
5. 随机推荐
6. 日历
7. 音乐播放器
8. 时钟
9. 移动导航

## 后台页面与交互

`/admin/settings` 包含四个标签页：

1. 网站设置
2. 社交链接
3. 首页布局
4. 主题修改

“主题修改”第一阶段只展示当前主题说明和“后续开放”提示，不允许编辑。

所有标签页共享一份未发布编辑状态：

- 点击“保存并发布”统一提交全部设置。
- 存在未保存修改时，离开页面前提醒管理员选择继续编辑或放弃修改。
- 发布成功后留在当前页面并显示短暂成功提示。
- 发布失败时保留当前编辑内容并显示具体错误。
- 不增加独立预览页，首页布局编辑器本身提供近似预览。

## 公开首页

公开首页服务端读取已发布的网站设置、启用的社交链接和布局数据：

- 网站标题和描述用于页面 Metadata。
- Favicon 用作浏览器标签页图标。
- 头像、用户名和状态短语同步到首页个人区域。
- 备案信息仅在填写后显示。
- 被关闭的可选模块不渲染，也不保留空白位置。
- 核心模块始终渲染。
- 桌面布局异常时忽略异常数据并使用默认位置。

图片加载失败时，公开页面显示默认头像或默认图标。数据库读取失败时，首页仍使用项目内默认内容正常显示。

## 校验与安全

发布前由服务端统一校验：

- 网站标题和用户名不能为空。
- 文本字段符合长度限制。
- 头像与 Favicon 是有效项目本地路径或 HTTPS URL。
- 社交链接仅使用 `https://` 或 `mailto:`。
- 布局模块标识已定义。
- 卡片位置在画布内且互不重叠。
- 核心模块不能关闭。
- 社交链接排序值不冲突。

安全边界：

- 只有已登录管理员可以访问 `/admin/settings` 和发布设置。
- 浏览器不能直接修改配置表。
- 数据库写入由服务端使用受保护的密钥完成。
- 公开首页只能读取已发布配置。
- 错误响应不泄露数据库或密钥信息。

## 错误处理

- 未登录访问后台设置页时跳转登录页。
- 权限不足时拒绝发布。
- 字段校验失败时指出具体字段并保留编辑内容。
- 网络或数据库失败时允许重试，且不改变公开首页。
- 公开首页读取异常时回退项目默认内容和布局。

## 验收标准

1. 管理员可以修改网站信息并统一发布。
2. 社交链接可以新增、编辑、排序、启用和删除。
3. 布局卡片可以拖拽、吸附网格，且无法重叠或越界。
4. 核心模块无法关闭；可选模块关闭后不在首页显示。
5. 恢复默认布局不会在发布前影响公开首页。
6. 未保存修改时离开页面会收到提醒。
7. 发布失败不会产生部分更新。
8. 公开首页正确读取设置，并在异常时回退默认内容。
9. 桌面端布局和移动端固定顺序通过外部浏览器验收。
10. `npm test`、`npm run lint`、`npx tsc --noEmit` 通过。
11. 为减少内存压力，生产构建只在最终验收阶段运行一次。

## 明确不做

- 不安装新依赖，不使用 Docker。
- 不提供图片上传或 Supabase Storage 管理。
- 不允许修改导航结构。
- 不提供自由缩放、卡片重叠或移动端拖拽。
- 不提供主题颜色编辑。
- 不保存历史发布版本。
