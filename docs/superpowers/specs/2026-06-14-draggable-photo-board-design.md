# 可拖拽照片画板与设置同步设计

## 背景

流程 5A 的真实照片上传、状态管理与公开相册已经完成本地验证。真实浏览器验收发现两个需要调整的问题：

- 后台照片设置保存成功且数据库已更新，但当前表单仍显示保存前的状态；
- 公开相册需要从中心堆叠改为可自由拖拽、分散铺放的照片画板。

## 范围

本变更只覆盖：

- 修复照片设置表单在保存成功后的状态与排序显示同步；
- 将 `/album` 当前组照片改为当前会话内可自由拖拽的画板；
- 简化照片灯箱为单张查看。

本变更不新增依赖、数据库字段、迁移或持久化布局。

## 后台设置同步

照片设置保存成功后，数据库仍是状态与排序值的唯一事实来源。

- Server Action 保存成功后继续重新验证相关路径；
- 客户端编辑器在收到新的服务端 `photo` 数据时重建设置表单；
- 状态选择框和排序输入框立即显示数据库返回的最新值；
- 保存失败时保留当前输入与错误提示，不伪装成已同步。

## 公开照片画板

### 初始布局

- 每组最多展示 `12` 张公开照片；
- 每次进入页面、刷新页面或切换分组时，为当前组生成新的随机位置；
- 照片分散铺放在画板的多个区域，不集中于中心；
- 允许照片自然重叠；
- 初始位置不得让照片完全离开画板。

随机布局只属于当前页面会话，不写入数据库或浏览器存储。

### 拖拽

- 鼠标与触控均可拖拽照片；
- 开始拖动时，该照片立即成为最高层；
- 拖动结束后，照片在当前页面会话内保留当前位置；
- 最后被拖动的照片始终位于最上层；
- 拖动位置受画板边界限制，照片不能完全拖出画板；
- 拖拽时不触发页面横向滚动；
- 小幅指针移动视为单击，不误判为拖拽。

### 点击与灯箱

- 单击未发生拖动的照片，直接打开该照片的灯箱；
- 灯箱只显示当前照片；
- 不显示关闭、上一张、下一张或页码按钮；
- 点击照片周围空白区域关闭灯箱并返回原画板位置；
- 保留 `Escape` 关闭能力，避免键盘用户被困在对话框中；
- 关闭后焦点返回触发照片。

上一组与下一组导航继续保留。

## 组件边界

- `src/lib/photos/board.ts`：负责纯函数布局生成、边界限制与拖拽距离判断；
- `src/components/photos/public-album.tsx`：负责当前组位置、层级、指针拖拽与打开灯箱；
- `src/components/photos/photo-lightbox.tsx`：负责单张照片灯箱、空白关闭和焦点恢复；
- `src/components/admin/photos/photo-editor.tsx`：负责设置表单随最新服务端照片重新初始化。

## 错误与可访问性

- 图片加载失败继续使用现有受控占位；
- 拖拽照片仍使用可聚焦语义控件；
- 单击、`Enter` 和 `Space` 均可打开灯箱；
- 灯箱保留 `aria-modal`、`Escape` 关闭和焦点恢复；
- `prefers-reduced-motion` 继续减少动画；
- 画板在约 `320px` 宽度不得造成页面级横向滚动。

## 验证标准

- 保存状态或排序后，设置表单立即显示真实新值；
- 刷新或切换组时，照片分散生成在画板内；
- 照片可用鼠标和触控拖动，且最后拖动者位于最上层；
- 单击照片打开无按钮的单张灯箱；
- 点击灯箱空白区域或按 `Escape` 返回画板；
- `/album` 桌面与约 `320px` 移动宽度无页面级横向滚动；
- `npm test`、`npm run lint`、`npx tsc --noEmit`、`npm run build -- --webpack` 和 `git diff --check` 通过。
