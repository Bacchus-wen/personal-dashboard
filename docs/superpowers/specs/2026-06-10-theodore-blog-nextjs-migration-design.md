# Theodore 博客 Next.js 迁移设计

## 目标

将 `prototype/` 中的 Theodore 原生 HTML、CSS 和 JavaScript 原型迁移为当前根目录下的 Next.js 正式网站，同时保留原型作为后续设计更新的参考源。

迁移结果需要：

- 尽可能还原原型的布局、色彩、字体、间距、圆角、阴影、动效和响应式行为。
- 将共享结构拆分为可复用 React 组件，避免复制页面 HTML。
- 将基础视觉规范整理为统一设计令牌。
- 修复原型中现有的中文乱码，使用清晰、自然的中文文案。
- 更新 `AGENTS.md`，说明项目结构、设计系统、组件规范和原型同步流程。
- 使用 Git 记录基线、迁移和后续设计同步。

## 项目边界

### 设计源

`prototype/` 是 Agent 生成的原型和视觉参考源。后续 Agent 可以继续更新该目录，但不直接覆盖正式网站代码。

### 正式网站

`src/` 是正式 Next.js 实现。原型更新后，需要先通过 Git 查看 `prototype/` 的变化，再选择性同步到 `src/`。

### 不在本次范围

- 不接入数据库、CMS、登录或真实编辑后台。
- 不实现真实音乐播放、内容发布或密钥导入。
- 不添加原型中没有要求的服务端 API。
- 不删除 `prototype/`。

### 组件职责

- `chrome/`：分页顶部导航、浮动工具、页面标题等跨页面结构。
- `home/`：首页侧边栏、欢迎卡片、相册预览、时钟、日历和播放器。
- `ui/`：玻璃卡片、标签、筛选器、搜索栏、内容卡片等基础复用组件。
- `data/site-content.ts`：文章、项目、资源和博客卡片的展示数据。
- `styles/design-tokens.css`：颜色、字体、间距、圆角、阴影、动效和断点变量。

## 设计系统

### 视觉方向

保持原型的轻盈个人仪表盘风格：

- 浅灰雾面背景和低饱和径向光晕。
- 半透明玻璃表面、细白边框和低对比阴影。
- 薄荷青用于主要交互，青蓝用于次级状态，暖黄和浅粉仅作点缀。
- 圆润但克制的卡片、按钮和图标。

### 核心设计令牌

保留并整理原型的基础颜色：

```css
--color-bg: #f2f5f4;
--color-surface: #ffffff;
--color-fg: #29484a;
--color-muted: #71868b;
--color-accent: #35c7b0;
--color-accent-secondary: #72b8c7;
--color-warm: #e7dc76;
--color-pink: #eaa6b9;
```

同时提取：

- 间距阶梯：页面安全边距、卡片内边距和网格间距。
- 圆角阶梯：按钮、常规卡片、大卡片和胶囊标签。
- 阴影：普通玻璃表面与悬浮状态。
- 动效：普通交互、导航滑块和页面进入动画。
- 响应式断点：`1180px`、`820px` 和 `520px`。

## 页面结构

### 首页

桌面端保持三列错落仪表盘：

- 左侧个人资料和页面导航。
- 中间相册预览、欢迎区、社交链接与内容推荐。
- 右侧时钟、日历和播放器。

窄屏时恢复正常文档流，并按欢迎区、社交链接、相册、导航、组件和内容的顺序重排。

### 内容页

所有内容页共享：

- 固定玻璃顶部导航。
- 当前页面 active 状态和悬浮滑动指示器。
- 右侧浮动工具。
- 居中的页面标题区。

页面主体分别使用：

- 文章：年份分组时间线。
- 项目：两列项目卡片。
- 资源与博客：可搜索和筛选的内容卡片网格。
- 相册：桌面端错落拍立得，移动端规则网格。
- 关于：高可读的单张玻璃内容卡片。

## 交互

需要保留的交互：

- 根据本地时间更新首页问候语和时钟。
- 顶部导航悬浮指示器。
- 浮动工具切换安静背景和返回顶部。
- 资源与博客的前端搜索及分类筛选。
- 播放器的演示播放状态。
- 点击相册照片后提升层级。
- 所有按钮、卡片、输入框具备 hover、active 和 focus 状态。

交互组件使用客户端组件；静态页面和内容数据优先保持服务端组件。

## 响应式与可访问性

- 最小支持宽度为 `320px`。
- 不允许页面整体横向滚动。
- 移动端触控目标不小于约 `44px`。
- 导航和图标按钮提供明确的 `aria-label`。
- 使用语义化的链接、按钮、标题和区域结构。
- 输入框提供可见 focus 状态。
- 支持 `prefers-reduced-motion`，减少动画而不影响功能。

## 内容与乱码修复

原型文件中的中文文本存在编码乱码。迁移时不复制乱码文本，而根据页面语义恢复自然中文：

- 保留 Theodore、英文眉题和模块名称。
- 中文标题和说明使用原型表达的含义重新整理。
- 缺失数据继续使用短横线，不虚构访问量等指标。

## Git 与原型同步流程

1. 正式迁移前提交当前项目和 `prototype/` 基线。
2. 迁移过程按设计系统、共享组件、页面和文档分阶段提交。
3. Agent 后续只更新 `prototype/`。
4. 使用 `git diff -- prototype` 审查新设计变化。
5. 将确认后的变化同步到 `src/`，完成验证后单独提交。

## 验证标准

- `npm run lint` 无错误。
- `npm run build` 成功。
- 所有目标路由可访问。
- 搜索、筛选、导航、背景切换、返回顶部、播放器和相册层级交互可用。
- 使用浏览器对照原型检查桌面端与移动端布局。
- `AGENTS.md` 明确记录结构、设计令牌、复用规则和同步流程。
