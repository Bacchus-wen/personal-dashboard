# 近日规划功能设计

## 目标

为 Theodore 个人工作台增加一套真实可用的“近日规划”功能。站长可以在受保护的后台创建、编辑、发布、暂停、完成、取消和删除规划；访客只能看到明确公开的有效规划。

第一阶段保持个人网站所需的中等复杂度：提供分类、筛选、分页、Markdown 详情、首页预览和回收站，但不加入子任务、看板、通知、批量操作或版本历史。

## 范围

### 第一阶段包含

- 后台规划列表、新建、编辑和回收站。
- 草稿、私密、公开三种可见性。
- 未开始、进行中、已暂停、已完成、已取消五种状态。
- 高、中、低三级优先级。
- 一个可选分类、可选截止日期和可选关联链接。
- 必填摘要与可选 Markdown 详细说明。
- 公开 `/plans` 页面及首页近日规划卡片。
- 服务端读取、管理员写入、输入校验和缓存刷新。

### 第一阶段不包含

- 邮件、浏览器或后台弹窗形式的截止提醒。
- 子任务、看板、拖拽和批量操作。
- 修改历史和旧版本恢复。
- 自动清理回收站。
- 多个关联链接。
- 分类自定义颜色。

## 技术方案

采用 Next.js 服务端安全架构与 Server Actions：

- 浏览器不能直接访问规划业务表。
- 公开页面由 Next.js 服务端读取并筛选公开数据。
- 后台 Server Actions 在每次写入前验证唯一管理员身份。
- 服务端使用 Supabase Secret Key 访问数据库。
- 成功写入后主动刷新首页和 `/plans` 的缓存。

不为该功能增加浏览器直连 Supabase、复杂业务 RLS、Docker 或 pgTAP。

## 数据模型

### `plan_categories`

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| `id` | UUID | 主键 |
| `name` | Text | 去除首尾空白后非空，最多 20 个字符，不区分大小写唯一 |
| `created_at` | Timestamptz | 创建时间 |
| `updated_at` | Timestamptz | 最后修改时间 |

删除分类时，相关规划的 `category_id` 自动变为 `null`。公开界面将其显示为“未分类”。

### `plans`

| 字段 | 类型 | 规则 |
| --- | --- | --- |
| `id` | UUID | 主键 |
| `title` | Text / Null | 草稿可为空；私密或公开时必填 |
| `summary` | Text / Null | 简短纯文本摘要；草稿可为空；私密或公开时必填 |
| `description` | Text / Null | 可选 Markdown 详细说明 |
| `status` | Enum | `not_started`、`in_progress`、`paused`、`completed`、`cancelled` |
| `visibility` | Enum | `draft`、`private`、`public` |
| `priority` | Enum | `high`、`medium`、`low` |
| `progress` | Integer | `0` 至 `100` |
| `deadline` | Date / Null | 可选，只保存日期 |
| `related_url` | Text / Null | 可选安全站内路径或 HTTP(S) 地址 |
| `category_id` | UUID / Null | 可选，外键删除时设为 `null` |
| `deleted_at` | Timestamptz / Null | 有值表示位于回收站 |
| `created_at` | Timestamptz | 创建时间 |
| `updated_at` | Timestamptz | 最后修改时间 |

新规划默认值：

- `visibility = draft`
- `status = not_started`
- `priority = medium`
- `progress = 0`

草稿允许标题、摘要等内容不完整。后台列表使用“未命名规划”作为空标题的显示占位，不将占位文字写入数据库。

## 业务规则

### 状态与进度

- 未开始：保存时自动将进度修正为 `0`。
- 进行中：进度必须为 `1` 至 `99`。
- 已暂停：保留当前进度，可为 `0` 至 `100`。
- 已完成：保存时自动将进度修正为 `100`。
- 已取消：保留当前进度，可为 `0` 至 `100`。

状态和进度在界面中分别编辑，但服务端保存时执行上述修正规则。

### 可见性与完整性

- 草稿仅管理员可见，允许字段不完整。
- 私密仅管理员可见，但标题、摘要、状态、优先级和进度必须有效。
- 公开内容必须满足与私密相同的完整性要求。
- 转为私密或公开时，服务端拒绝缺少必填内容的保存请求，并返回明确字段错误。

### 公开规则

公开 `/plans` 只展示满足全部条件的规划：

- `visibility = public`
- `deleted_at is null`
- 状态为未开始、进行中或已暂停

首页近日规划只从以下规划中选择：

- 满足公开规则
- 状态为未开始或进行中
- 设置了截止日期

首页候选项先展示逾期规划，再展示尚未截止的规划。多个逾期规划中，最近逾期的一条优先；多个尚未截止的规划中，截止日期最近的一条优先。若无候选项，首页卡片保持原位置并显示“近期暂无公开规划”。

### 排序

公开 `/plans` 的排序规则：

1. 优先级：高、中、低。
2. 同优先级按截止日期从近到远。
3. 无截止日期排在同优先级最后。
4. 日期相同时按最后修改时间从新到旧。

后台默认按最后修改时间从新到旧排列。筛选和搜索不改变默认排序。

### 日期标签

截止日期只保存日期，不保存具体时间。访客浏览器根据自己的本地日期计算标签：

- 截止日已过去：红色“已逾期 X 天”。
- 截止日为今天：红色“今天截止”。
- 截止日在未来 1 至 3 天：橙色“剩余 X 天”。
- 截止日在未来超过 3 天：显示普通日期。
- 未设置截止日期：显示“暂未设定截止日期”。

标签同时使用文字和颜色。不同地区访客可能在日期交界时看到不同标签，这是预期行为。服务端排序只比较存储的日期值，不依赖访客时区。

### 关联链接

- 支持以 `/` 开头的安全站内相对路径。
- 支持 `http://` 和 `https://` 外部地址。
- 拒绝脚本协议、协议相对地址及其他不受支持的格式。
- 站内链接在当前页面打开。
- 外部链接在新标签页打开，并使用外链标识以及安全的 `rel` 属性。

### 删除与恢复

- 普通删除只写入 `deleted_at`，记录进入回收站并立即停止公开展示。
- 回收站不自动永久删除记录。
- 恢复时清除 `deleted_at`，保留业务内容，并强制将可见性改为草稿。
- 永久删除前显示“无法恢复”的二次确认弹窗。
- 永久删除失败时保留记录并显示错误。

## 页面与交互

### 公开 `/plans`

- 每页展示 9 条规划，使用分页按钮。
- 支持状态、优先级和分类组合筛选。
- 筛选条件与页码记录在 URL 查询参数中。
- 更改筛选条件后返回第一页。
- 卡片默认折叠，展示标题、摘要、分类、状态、优先级、进度和截止日期。
- 点击“展开详情”后在卡片内部渲染 Markdown 详细说明和关联链接。
- 展开后可点击“收起详情”。
- 页面初次加载时所有卡片保持折叠。
- 读取失败时，仅规划区域显示“规划暂时无法加载，请稍后重试”，网站其他部分继续工作。

公开状态筛选只包含未开始、进行中和已暂停。已完成、已取消、草稿、私密及回收站记录不会发送到访客浏览器。

### 首页近日规划卡片

- 替换现有首页底部的小型演示模块。
- 显示标题、状态、进度、截止日期标签和分类。
- 桌面端悬浮约 300 毫秒后显示较大的预览浮层。
- 鼠标从卡片移入浮层时，浮层保持显示；离开卡片与浮层区域后关闭。
- 浮层显示摘要、Markdown 转换后的安全纯文本预览、状态、进度、优先级、截止日期和关联链接。
- 移动端首次点击卡片时打开预览弹窗；弹窗支持关闭按钮、点击遮罩和返回键关闭。
- 桌面端点击原卡片直接跳转到 `/plans`；桌面预览和移动端弹窗提供“查看全部规划”入口。

### 后台规划列表 `/admin/plans`

- 每页展示 12 条卡片，使用分页按钮。
- 支持标题搜索、状态、可见性、优先级和是否逾期组合筛选。
- 查询条件和页码记录在 URL 中；更改条件后返回第一页。
- 每张卡片展示标题或“未命名规划”、状态、可见性、优先级、进度、分类和截止日期。
- 提供新建规划、编辑规划、移入回收站、进入回收站和管理分类入口。
- 第一阶段每次只操作一条规划，不提供批量操作。

### 新建与编辑

- `/admin/plans/new` 用于创建规划。
- `/admin/plans/[id]/edit` 用于编辑规划。
- 使用手动保存，不自动保存。
- 保存期间禁用重复提交。
- 保存成功后留在当前页面，显示短暂成功提示。
- 保存失败时保留表单内容，并显示字段错误或通用错误。
- 页面有未保存修改时，站内导航或关闭页面前提醒用户。
- 编辑器包括纯文本摘要字段和 Markdown 详细说明字段。
- Markdown 编辑器提供编辑和预览能力。

### 分类管理

规划列表页提供“管理分类”弹窗：

- 新增、重命名和删除分类。
- 分类名称去除首尾空白后不可为空、不可重复，最多 20 个字符。
- 删除分类前提示受影响的规划数量。
- 删除后相关规划自动变为“未分类”。
- 分类颜色第一阶段由系统从现有设计系统中自动分配低调颜色，不持久化自定义颜色。

### 回收站 `/admin/plans/trash`

- 展示已删除规划及删除时间。
- 支持逐条恢复和永久删除。
- 恢复后规划转为草稿。
- 永久删除使用明确的二次确认弹窗。

## 组件边界

- 路由页面保持以服务端组件为主，负责查询、权限检查和组合布局。
- 筛选器、分页、展开卡片、悬浮预览、移动弹窗、Markdown 编辑器和未保存提醒使用聚焦的客户端组件。
- 规划查询、写入、校验、日期计算和 URL 校验分别放在职责清晰的服务端或共享模块中。
- 后台表单复用同一个规划编辑组件，新建与编辑路由只负责提供初始数据。
- 公开规划卡片与后台规划卡片分别实现，避免把权限操作混入公开组件。
- 视觉实现复用现有设计变量及 `.glass`、`.card`、`.pill`、`.btn` 等样式约定。

## 数据流与缓存

### 公开读取

1. Next.js 服务端读取公开规划和公开筛选所需分类。
2. 服务端应用公开条件、排序与分页。
3. 服务端仅向页面组件提供公开展示所需字段。
4. 客户端仅负责筛选 URL、展开状态和本地日期标签。

### 后台写入

1. 管理员提交 Server Action。
2. Server Action 验证当前登录用户等于 `ADMIN_USER_ID`。
3. 服务端校验、规范化并修正输入。
4. 服务端使用 Secret Key 执行数据库写入。
5. 成功后刷新首页、`/plans` 和相关后台路径缓存。
6. 返回成功或结构化错误结果。

创建、编辑、删除、恢复、永久删除、分类修改和可见性调整成功后，都必须刷新受影响缓存，使访客刷新页面即可看到最新内容。

## 安全边界

- `plans` 和 `plan_categories` 启用 RLS。
- 撤销 `anon` 与普通 `authenticated` 角色对表及相关序列的直接权限。
- 只授予 `service_role` 必需权限。
- 所有后台 Server Actions 在写入前重新验证管理员身份。
- 不信任浏览器提交的状态、可见性、进度、分页或筛选值。
- Markdown 渲染必须过滤危险 HTML 和脚本；第一阶段可以完全禁用 Markdown 内嵌 HTML。
- 所有关联链接都经过服务端 URL 校验。
- 错误响应不包含 Secret Key、数据库连接信息或内部堆栈。
- `.env.local` 和真实密钥继续保持 Git 忽略状态。

## 错误处理

- 公开查询失败：规划区域显示可恢复错误状态，不影响其他页面模块。
- 后台列表查询失败：显示错误状态和重试入口，不伪装为空列表。
- 保存失败：保留表单内容，显示字段级或页面级错误。
- 权限失败：不执行写入，跳转登录或返回无权限错误。
- 删除、恢复或分类变更失败：保持原数据状态并明确提示。
- 空结果：公开页、后台筛选结果和回收站分别显示合适的空状态。

## 验证策略

### 自动化验证

- 数据校验测试：
  - 状态与进度自动修正。
  - 草稿允许不完整，私密与公开拒绝不完整内容。
  - 分类名称规范化、长度限制和去重。
  - 安全站内路径、HTTP(S) 地址及危险链接拒绝。
- 查询规则测试：
  - 公开内容不会包含草稿、私密、已完成、已取消或回收站记录。
  - 首页候选选择、优先级排序、日期排序和无截止日期排序。
  - 筛选与分页参数规范化。
- 权限测试：
  - 未登录用户和非管理员不能调用写入操作。
  - 浏览器角色不能直接读取或写入业务表。
- UI 行为测试：
  - 状态与进度联动。
  - Markdown 预览安全渲染。
  - 未保存离开提醒。
  - 卡片展开、筛选和分页。

### 手动验证

- 桌面端首页悬浮预览的延迟、保持和关闭行为。
- 移动端首页预览弹窗和关闭行为。
- `/plans` 与后台页面在桌面和 320px 宽度下无横向滚动。
- 创建、编辑、公开、转私密、移入回收站、恢复和永久删除完整流程。
- 数据库故障时公开页面和后台错误状态。
- `npm test`、`npm run lint`、`npm run build`。

## 依赖与安装边界

Markdown 编辑、渲染和安全过滤可能需要新增小型 npm 依赖。安装任何依赖前，必须先向用户说明：

- 依赖名称与用途。
- 预计磁盘占用。
- 默认缓存位置及是否可能写入 C 盘。
- 不安装依赖的替代方案。

获得用户明确批准后，才可使用项目本地、优先位于 F 盘的 npm 缓存执行安装。
