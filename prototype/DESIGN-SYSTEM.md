# Theodore 个人博客设计系统

版本：1.0  
适用范围：首页、相册、近期文章、项目、关于、资源、优秀博客页面  
实现基准：`shared.css`、`shared.js`

## 1. 设计定位

Theodore 是一个轻盈、柔和、具有个人仪表盘感的博客网站。视觉表达以浅色雾面背景和玻璃拟态组件为核心，强调清晰层级、舒适留白与克制的薄荷青交互反馈。

核心关键词：

- 清新、柔和、干净、轻量
- 可爱但不过度幼稚
- 浅色玻璃拟态
- 低饱和渐变光晕
- 圆润、克制、具有个人感

## 2. 设计原则

### 2.1 内容优先

- 每个模块只承担一个主要目的。
- 标题、状态、数据和操作必须有明确层级。
- 不使用无意义装饰、虚构数据或冗余图标。

### 2.2 轻盈层级

- 使用透明度、模糊、细描边和低对比阴影建立层级。
- 避免纯白实心大面积容器和高对比黑色阴影。
- 页面强调色只用于 active 状态、关键操作和少量反馈。

### 2.3 一致交互

- 可点击卡片悬浮时轻微上浮。
- 首页主要模块悬浮时可同时轻微放大。
- 图标默认低饱和，悬浮或 active 时切换为薄荷青。
- 动画应自然、快速，不影响内容读取。

## 3. 基础设计令牌

所有页面必须复用 `shared.css` 中的根级令牌，不在单个页面中重复定义颜色。

```css
:root {
  --bg: #f2f5f4;
  --surface: #ffffff;
  --fg: #29484a;
  --muted: #71868b;
  --border: #ffffffb8;
  --accent: #35c7b0;
  --accent-2: #72b8c7;
  --warm: #e7dc76;
  --pink: #eaa6b9;
  --shadow: #50647816;

  --font-display: "Avenir Next", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif;
  --font-body: "Nunito", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif;
  --font-mono: "SFMono-Regular", Consolas, monospace;
}
```

### 3.1 色彩职责

| 令牌 | 用途 |
| --- | --- |
| `--bg` | 页面浅灰雾面基础背景 |
| `--surface` | 卡片、按钮和高亮表面 |
| `--fg` | 标题、主要正文和高对比信息 |
| `--muted` | 次级文字、说明和默认图标 |
| `--border` | 玻璃卡片细描边 |
| `--accent` | active、主操作、悬浮图标和当前日期 |
| `--accent-2` | 次级状态、播放器和引用块 |
| `--warm` | 星级、头像底色和暖色光晕 |
| `--pink` | 悬浮工具默认图标色 |
| `--shadow` | 所有低对比阴影 |

### 3.2 强调色预算

- 单个模块优先只使用一种强调色。
- 薄荷青用于主要交互；青蓝用于次级状态。
- 黄色仅用于评分或少量暖色提示。
- 粉色仅用于轻量工具图标，不作为主要操作色。

## 4. 背景系统

默认页面背景由浅灰底色与三组低饱和径向光晕组成：

- 左下：薄荷绿
- 中下：淡黄色
- 右下：青蓝色

背景固定于视口，不随内容滚动。需要更安静的阅读场景时，可为 `body` 添加 `.quiet`，降低光晕饱和度和数量。

禁止使用：

- 硬边渐变
- 高饱和大面积色块
- 紫色科技渐变
- 暖米色、橙棕色作为全页底色

## 5. 字体与排版

### 5.1 字体职责

- Display：页面标题、卡片标题、品牌名称。
- Body：正文、按钮和说明。
- Mono：时间、日期、Views、Marks、眉题和数据。

### 5.2 排版规范

| 类型 | 建议规格 |
| --- | --- |
| 页面主标题 | `clamp(34px, 4vw, 58px)`，紧凑字距 |
| 首页问候标题 | `clamp(32px, 5vw, 52px)` |
| 卡片标题 | 14–22px，根据模块层级选择 |
| 正文 | 16px / 1.55 |
| 次级说明 | 12–14px |
| 眉题 | 11px Mono、大写、`0.14em` 字距 |
| 数据 | Mono，并启用等宽数字 |

文字不使用纯黑。标题使用 `--fg`，次级内容使用 `--muted`。

## 6. 圆角、间距与阴影

### 6.1 圆角

| 组件 | 圆角 |
| --- | --- |
| 大型玻璃卡片 | 30–36px |
| 常规卡片 | 26–30px |
| 导航容器 | 24–32px |
| 按钮与输入框 | 13–15px |
| 标签 | 999px |
| 圆形工具按钮 | 50% |

### 6.2 间距

- 页面最大内容宽度：1280px。
- 首页最大内容宽度：1420px。
- 常规卡片内边距：28px。
- 卡片网格间距：20–24px。
- 页面左右安全边距：桌面 28px 以上，手机 14px 以上。

### 6.3 阴影

阴影必须保持柔和且低对比：

```css
box-shadow:
  0 18px 60px var(--shadow),
  inset 0 1px color-mix(in oklch, var(--surface) 74%, transparent);
```

禁止使用高对比黑色阴影或多层厚重悬浮效果。

## 7. 玻璃拟态容器

`.glass` 是全站基础表面类，必须用于主要卡片、导航、搜索和悬浮工具。

```css
.glass {
  background: color-mix(in oklch, var(--surface) 64%, transparent);
  border: 1px solid var(--border);
  box-shadow: 0 18px 60px var(--shadow);
  backdrop-filter: blur(22px) saturate(125%);
}
```

使用规则：

- 内容密集区域可提高表面不透明度。
- 阅读卡片必须保证文字对比度。
- 不在玻璃卡片内部重复堆叠多个同强度玻璃层。

## 8. 图标系统

### 8.1 视觉语言

- 使用圆润线性 SVG 图标。
- 默认采用 `currentColor`，不在 SVG 内写死颜色。
- 统一使用圆角线帽和圆角连接。
- 常规描边宽度为 `1.8–1.9`。

### 8.2 导航图标状态

| 状态 | 图标颜色 | 背景与动效 |
| --- | --- | --- |
| 默认 | `--muted` | 无独立高亮 |
| Hover | `--accent` | 淡薄荷底板，轻微放大 |
| Active | `--accent` | 滑动圆形指示器停留 |

左侧首页菜单图标悬浮时使用淡青圆角底板；分页顶部导航使用滑动圆形指示器。图标必须保留清晰语义，不使用通用装饰图标替代页面含义。

## 9. 组件规范

### 9.1 卡片

- 基础组合：`.glass.card`。
- 可点击卡片增加 `.lift`。
- 首页主要模块悬浮：上浮 `6px` 并放大至 `1.018`。
- 其他页面可点击卡片悬浮：仅上浮 `6px`。
- 社交链接按钮不参与首页模块整体放大规则。

### 9.2 按钮

- 默认按钮使用半透明白色表面和细描边。
- 主按钮使用 `--accent` 背景与白色文字。
- Hover 时上浮约 `2px`，不得出现剧烈缩放。
- 所有按钮最小高度应接近或高于 42px。

### 9.3 标签与筛选器

- 使用胶囊形状。
- 默认状态：半透明浅色背景与 `--muted` 文字。
- Hover：背景变亮。
- Active：`--accent` 背景与白色文字。

### 9.4 搜索框

- 长条圆角输入框，居中展示。
- Focus 时边框切换为 `--accent`。
- Focus 光环使用低透明薄荷青，不使用浏览器默认蓝色描边。

### 9.5 导航

首页使用左侧竖向菜单；分页使用左上角横向图标导航。

- 当前页必须显示 active 状态。
- 分页导航悬浮指示器应平滑滑动，移出后返回 active 项。
- 手机端分页导航允许横向滚动，但不得造成页面整体横向滚动。

### 9.6 悬浮工具

- 固定在页面右侧中部。
- 默认图标使用浅粉色。
- Hover 时变为薄荷青并放大至约 `1.09`。
- 手机窄屏下隐藏，避免遮挡内容。

## 10. 页面级模式

### 10.1 首页仪表盘

- 桌面端采用左侧导航、中间内容、右侧组件的错落式布局。
- 整体视觉重心位于视口中部偏下。
- 主要模块可轻微放大；社交按钮保持独立主题色滑入效果。
- 手机端按欢迎、社交、相册、导航、组件、内容卡片顺序重排。

### 10.2 内容网格

- 项目页桌面两列。
- 资源与博客页桌面三列。
- 1180px 以下三列变两列。
- 820px 以下统一单列。

### 10.3 文章时间线

- 日期、圆点、标题、标签必须对齐。
- 竖向虚线保持低对比。
- Hover 时文章行轻微右移，标题切换为薄荷青。

### 10.4 相册照片

- 使用白色拍立得边框、自然旋转和错落层级。
- Hover 时照片上浮、归正、放大并提高视觉层级。
- 手机端改为规整单列或双列，不保留绝对定位堆叠。

## 11. 动效规范

全站动效必须轻量、可预测。

| 场景 | 时长 | 建议缓动 |
| --- | --- | --- |
| 常规 Hover | 200–250ms | `ease` |
| 导航滑动指示器 | 320ms | `cubic-bezier(.22, .8, .22, 1)` |
| 页面进入 | 550ms | `ease` |
| 卡片进入 | 550ms | `ease` |

限制：

- 不使用持续循环的装饰动画。
- 不让 Hover 动效导致布局跳动。
- 不同时叠加明显位移、缩放、旋转和颜色变化。

## 12. 响应式规范

### 12.1 关键断点

| 断点 | 行为 |
| --- | --- |
| `> 1180px` | 完整桌面仪表盘、三列内容网格 |
| `≤ 1180px` | 首页变为两列，右侧组件横向排列；内容网格两列 |
| `≤ 820px` | 页面单列，首页恢复正常文档流，分页导航可横向滚动 |
| `≤ 520px` | 紧凑导航、双列社交按钮、隐藏右侧悬浮工具 |

### 12.2 基本要求

- 最小支持宽度为 320px。
- 不允许页面整体横向滚动。
- 卡片不使用阻碍重排的固定宽度。
- 移动端触控目标建议不低于 44px。
- 移动端不依赖 Hover 才能理解功能。

## 13. 内容与文案

- 以中文为主，允许保留自然的英文模块名和操作名。
- 英文使用统一大小写，如 `Good Morning`、`Website`、`Views`、`Marks`。
- 缺少真实数据时使用短横线 `—`，不虚构指标。
- 卡片标题保持简短，说明文字优先控制在两行内。

## 14. 可访问性与实现要求

- 图标按钮必须提供准确的 `aria-label` 或 `title`。
- 所有交互组件必须可通过链接或按钮语义访问。
- 使用 `currentColor` 保证图标状态与文字状态同步。
- 输入框必须保留清晰的 focus 状态。
- 动态问候、时间和筛选功能由共享脚本维护，避免页面间行为不一致。

## 15. 新组件接入清单

增加新页面或组件前，检查以下项目：

- [ ] 使用现有颜色令牌，没有新增无必要颜色。
- [ ] 使用 `.glass`、`.card`、`.lift` 或已有组件模式。
- [ ] 图标为圆润线性 SVG，并使用 `currentColor`。
- [ ] 默认、Hover、Active、Focus 状态明确。
- [ ] 强调色使用克制，没有在同一模块中竞争。
- [ ] 桌面、平板和手机布局均可用。
- [ ] 没有整体横向滚动。
- [ ] 文案具体且没有虚构数据。
- [ ] 导航 active 状态和页面跳转正确。
- [ ] 动效不会导致布局跳动或阻碍阅读。

## 16. 维护约定

- `shared.css` 是视觉规范的实现来源。
- `shared.js` 是共享交互的实现来源。
- 页面文件只负责页面内容和结构，不重复定义全局设计令牌。
- 修改颜色、圆角、阴影或共享状态时，应同步更新本说明文件。
- 若文档与代码不一致，以当前产品实现为准，并尽快修正文档。
